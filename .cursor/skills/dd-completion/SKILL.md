# DD 完了報告（ステップ9）

全ての作業が完了したら、チェックリストで完了状態を確認し、次のステップを明確化する。

## コマンド

- **コマンド**: `/dd:wrap`
- **実行**: このスキルの手順・チェックリストに従う

## DD完了チェック（/dd:wrap 実行時・対象は指定した案件ルート）

- [ ] ステップ2: 案件内 `業務整理用/` → `dd materials/` の分類・配置が完了
- [ ] ステップ2.5: 各フォルダのサマリが作成/更新済み
- [ ] ステップ3: 事業概要.md と 投資条件概要.md が更新済み
- [ ] ステップ4: ヒアリングリスト.md が更新済み
- [ ] ステップ5: 初期レビューが完了し、評価ファイルが作成/更新済み
- [ ] ステップ6: 投資仮説が更新済み
- [ ] ステップ7: research/ の追加調査がまとまっている
- [ ] ステップ8: 参考資料が適切に保存されている

## 週次チェック（対象は当該案件ルート）

- [ ] 新規資料の追加有無（案件内 業務整理用/ または dd materials/）
- [ ] 新規非mdファイルのサマリ追記
- [ ] 重要数値変化時の関連ファイル反映
- [ ] リスク項目.md への新論点追記
- [ ] 未確認/要ヒアリングの次アクション化

## 月次/決算チェック

- [ ] 月次試算表の最新化・サマリ反映
- [ ] 資金繰りの最新化（ランウェイ/前提）
- [ ] 税務申告の更新有無・重要論点
- [ ] 事業計画の更新有無・前提変更
- [ ] 投資条件概要.md の更新（valuation/調達額/資金使途）
- [ ] review_results/ への重要差分反映

## 完了報告に含める内容

1. **作業完了の確認**: 各ステップの完了状況、作成・更新したファイルの一覧
2. **主要な発見事項**: 重要な評価結果、主要な投資仮説、重要なリスク要因
3. **次のステップ**: 追加で必要な作業、ヒアリングで確認すべき事項、追加調査が必要なテーマ
