# Cursor エージェント指示書（このリポジトリでどう働くか）

## 目的

このリポジトリ（`DD_template`）は、デューデリジェンス（DD）の情報整理・レビュー・仮説構築を標準化するテンプレートです。  
エージェントは **「資料の配置 → サマリ作成 → 主要ファイル更新 → 初期レビュー → 仮説/リサーチ → 完了報告」** を、再現性高く支援します。

## 基本ルール

- **言語**: 出力（説明・文章・テンプレ）は日本語を基本とする
- **改変方針**: 既存のファイル構造・命名を尊重し、必要最小限の差分で更新する
- **根拠**: 断定せず、根拠となる資料（`dd materials/` や `research/` の参照先）を必ず併記する
- **不足情報**: 不足している情報は推測で埋めず、「未確認」「要ヒアリング」として明示する
- **更新の粒度**: 大量更新は避け、アクションごとに分割して進める（擬似コマンドを使う）

## リポジトリ内の“作業対象”の考え方

- **インプット（素材）**: `dd materials/` と `research/`
- **アウトプット（評価・仮説）**: `review_and_hypothesis/`
- **ガイド/運用**: `agents/`

## 擬似スラッシュコマンドの使い方（運用ルール）

このリポジトリでは、以下のように“擬似コマンド”を依頼文の先頭に付けてエージェントを呼び出します。

例:

- `/dd:init 会社名=XXX 対象期間=2024-2025` を実行して
- `/dd:summarize folder="dd materials/financial/月次試算表"` を実行して

コマンド仕様の一覧は `agents/cursor/commands.md` を参照。

## よくある成果物の品質基準

- **サマリ**: 「基本情報 / 内容サマリ / 主要ポイント / 投資判断への影響」が揃っている
- **レビュー**: 「概要 / 主要発見 / ポジ要因 / リスク / 推奨 / 結論 / 参照先」が揃っている
- **仮説**: 「投資テーゼ / 成長シナリオ / MOAT / 出口 / リスクと対策」が揃っている
