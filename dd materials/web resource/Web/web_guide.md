# Web記事スクレイピングガイド

ウェブ検索で得た参考になる記事をスクレイピングし、`Web`フォルダ内に格納する方法です。

---

## スクレイピングの目的

- DD作業中に参考になったWeb記事を体系的に保存
- 投資判断の根拠となる情報を記録
- 後から参照・検証できるように整理

---

## 保存先フォルダ構造

### 基本構造

```
web resource/Web/
├── web_guide.md (このファイル)
├── [カテゴリ名]/
│   ├── [記事タイトル].md
│   └── ...
└── ...
```

### カテゴリの例

必要に応じて以下のようなカテゴリフォルダを作成してください：

- `市場動向/`: 市場トレンド、業界動向に関する記事
- `競合分析/`: 競合企業に関する記事
- `M&A事例/`: M&A事例、買収事例に関する記事
- `技術動向/`: 技術トレンド、技術解説記事
- `財務分析/`: 財務分析手法、評価手法に関する記事
- `法務・規制/`: 法務、規制に関する記事
- `業界レポート/`: 業界レポート、調査報告書
- `その他/`: 上記に該当しない記事

---

## ファイル命名規則

### 基本ルール

1. **ファイル名**: `[記事タイトル].md` または `[YYYY-MM-DD]_[記事タイトル].md`
2. **文字制限**: ファイル名は50文字以内を推奨（長い場合は省略）
3. **使用可能文字**: 英数字、日本語、ハイフン、アンダースコア
4. **禁止文字**: スラッシュ、バックスラッシュ、特殊記号

### 命名例

```
市場動向/
├── 2025-01-15_AI市場の成長予測.md
├── SaaS業界の最新動向.md
└── 2025-01-20_スタートアップ投資動向レポート.md
```

## 保存する内容

各Markdownファイルには以下の情報を含めてください：

### 必須項目

1. **記事タイトル**: 元の記事のタイトル
2. **URL**: 記事のURL（出典として重要）
3. **公開日**: 記事の公開日（情報の鮮度を管理）
4. **取得日**: スクレイピングした日付
5. **記事本文**: 記事の主要な内容（要約または全文）

### 推奨項目

6. **カテゴリ**: 記事の分類
7. **関連性**: DD作業への関連性、投資判断への示唆
8. **主要なポイント**: 記事から得られた重要な情報（箇条書き）
9. **参考箇所**: 特に参考になった箇所の引用
10. **タグ**: 検索性を高めるためのタグ（例: #市場分析 #競合 #M&A）

## ファイルテンプレート

以下のテンプレートを使用してください：

```markdown
# [記事タイトル]

## 基本情報

- **URL**: [記事のURL]
- **公開日**: YYYY-MM-DD
- **取得日**: YYYY-MM-DD
- **カテゴリ**: [カテゴリ名]
- **タグ**: #[タグ1] #[タグ2]

## 記事要約

[記事の要約を記載]

## 主要なポイント

- [ポイント1]
- [ポイント2]
- [ポイント3]

## 記事本文

[記事の本文を記載]

## DD作業への関連性

[この記事がDD作業や投資判断にどのように関連するかを記載]

## 参考箇所

> [特に参考になった箇所の引用]

## その他メモ

[追加のメモや考察があれば記載]
```

## スクレイピングの手順

### ステップ1: 記事の選定

1. ウェブ検索で参考になる記事を見つける
2. 記事の信頼性を確認（出典、著者、公開日など）
3. DD作業への関連性を判断

### ステップ2: カテゴリの決定

1. 記事の内容に基づいて適切なカテゴリを選択
2. 該当するカテゴリフォルダがなければ作成
3. 複数のカテゴリに該当する場合は、最も関連性の高いカテゴリに保存

### ステップ3: ファイルの作成

1. 適切なファイル名でMarkdownファイルを作成
2. テンプレートに従って情報を記載
3. 記事の本文をスクレイピングして保存

### ステップ4: 内容の整理

1. 記事の要約を作成
2. 主要なポイントを抽出
3. DD作業への関連性を記載

## 注意事項

### 著作権

- **引用の明記**: 記事の出典（URL、著者名など）を必ず明記
- **引用範囲**: 必要最小限の範囲で引用する
- **全文転載の注意**: 全文を転載する場合は、著作権者の許可が必要な場合がある
- **個人利用**: 個人のDD作業用として使用することを前提とする

### 情報の信頼性

- **出典の確認**: 信頼できる情報源か確認する
- **公開日の確認**: 情報の鮮度を確認する
- **複数ソースの確認**: 重要な情報は複数のソースで確認する

### 情報の管理

- **定期的な整理**: 古い情報や不要な情報は定期的に整理する
- **重複の回避**: 同じ記事を複数回保存しないよう注意する
- **バックアップ**: 重要な情報はバックアップを取る

## 活用方法

### 1. 投資判断の根拠として

- 評価結果や投資仮説の根拠として参照
- リスク評価の参考資料として活用

### 2. リサーチの補完として

- `research`フォルダ内のリサーチ結果を補完
- 市場分析や競合分析の参考資料として活用

### 3. ヒアリングの準備として

- ヒアリングで確認すべき事項の参考として活用
- 質問項目の作成に活用
