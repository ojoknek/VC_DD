# 業務整理用（インポート資料の置き場）

案件ごとに受け取った資料を、**対応表で定めたフォルダ名**で整理し、ここに置く。  
`/dd:copy` または `program/run_copy.py` 実行で、中身が **dd materials/** の各フォルダへ変換・コピーされる。

---

## フォルダ構成の考え方

- **業務整理用** の直下に、**対応表の「業務整理用フォルダ」名どおりのサブフォルダ**を作る。
- 各サブフォルダの中に、該当する資料ファイル（PDF・XLSX・画像など）を入れる。
- フォルダ名は番号・スペース・文字を **対応表と完全に一致させる**（一致しないフォルダは処理対象外）。

## ワークフロー

1. **template を `deals/{会社名}/` にコピーしたあと**、当該案件の `業務整理用/`（この README があるフォルダのコピー先）で作業する。
2. 上記対応表に従い、**業務整理用の直下にフォルダを作成**し、中に資料を配置する。
3. **`/dd:copy` 実行**（案件ルートを指定）または **`python program/run_copy.py <案件ルート>`** を実行する。
   - **読み取り元**: 当該案件の `業務整理用/` の各サブフォルダ。
   - **保存先**: 当該案件の `dd materials/` の対応するフォルダ（`dd_logic/template/dd materials` と同じ構成）。
   - **変換**: PDF → ページごと PNG、XLSX/XLS → シートごと CSV。それ以外はそのままコピー。
   - **元ファイル**: `業務整理用/` に残る（バックアップ）。

詳細は以下を参照。

- 対応表・手順: `.cursor/skills/dd-copy-classify/SKILL.md`
- プログラム・保存先: リポジトリ直下の `program/README.md` および `program/run_copy.py` 内の `COPY_MAP`
